CloudClaim — Serverless Claims Pipeline
A fully serverless, event-driven healthcare claims processing pipeline built on AWS. Processes claims with zero data loss using SQS Dead Letter Queues, secured with HIPAA-aligned IAM policies and AES-256 encrypted S3 storage. Validated on 1,000+ simulated claims end-to-end.
1,000+ claims validated Zero data loss HIPAA-aligned AES-256 encrypted Full IaC (Terraform)
Key Highlights
Fault-tolerant via SQS Dead Letter Queues — no claim ever lost in 1,000+ simulated runs
SNS alerts on claim rejection with structured reason codes for downstream audit
REST API via API Gateway + Lambda for claim submission with schema validation
CloudWatch dashboards tracking Lambda errors, SQS queue depth, and p99 latency
Full Terraform IaC — entire AWS stack reproducible in a single command
Event-driven pipeline: S3 upload → Lambda Ingestor → SQS + DLQ → Lambda Processor → DynamoDB. API Gateway exposes REST query endpoints. SNS handles rejection notifications with structured payloads.
Data Flow
Claim JSON uploaded to S3 triggers Lambda Ingestor via S3 event notification
Ingestor validates schema and enqueues to SQS; failed messages routed to DLQ
Lambda Processor dequeues, applies business rules, writes result to DynamoDB
API Gateway + Lambda exposes GET endpoints for claim status queries
Fully provisioned via Terraform with GitHub Actions CI/CD. CloudWatch monitors all Lambda functions with custom dashboards, metric alarms, and structured log groups per service.
Infrastructure Stack
Terraform IaC — 100% reproducible deployment, no manual console steps
GitHub Actions CI/CD pipeline with lint, test, and auto-deploy on push to main
CloudWatch alarms + dashboards for full Lambda and SQS observability
IAM least-privilege roles scoped per Lambda function and service boundary
